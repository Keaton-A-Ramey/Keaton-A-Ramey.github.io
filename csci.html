Autonomous Drone Capstone Project

This is a 2-semester project I am doing with 3 Electrical Engineering Students. The overall goal is to have a system where a Raspberry Pi is connected to 3 (for now) cameras that transmit their image data to a central Pi. Using these 3 images, this central Pi will automatically use software that I have already written (3D localization software) to accurately calculate the real-world position of the drone.

Using that real-world position, this central Pi will decide on and issue instructions to the drone in order to move it along a predetermined path. The end goal is to fit hyperspectral cameras to this drone in order to use its images to analyze crop health in an indoor farm. These insights will be used to decide what nutrients and how much of them to give to areas of crops.

To be more specific, this semester I have delivered software that does 2 main things:

  1. Given the rough, 3-D locations of a camera and both the 3-D locations and pixel-coordinates of 5 objects in the image (we used LEDs), the software will run a least-squares optimization to very accurately calculate the location and orientation of the camera.
  2. Given the pixel coordinates of an object that appears in multiple images that I am able to run that least-squares optimization with, I can accurately calculate the approximate 3D location of said object. This is how we'll know where the drones are.

Next semester, I will be working on processes by which we can make those 2 things an automatic rather than a manual process, as I currently have to feed the program those coordinates. Additionally, I will likely be working on interfacing with the drone, writing software to control its movement.

Activity Classification Project

This project was done over the course of a semester in CSE 432/532: Machine Learning. It focused on classifying human activities using data from four sensors: accelerometer, gyroscope, magnetometer, and pressure sensor. The data came from 100+ experiments from a dozen users, and the goal was to preprocess it, extract features, and train machine learning models to identify activities like walking, running, and typing.

I started by organizing the raw sensor data, conducting exploratory data analysis (EDA), and cleaning it up. Missing values were handled using interpolation, filling techniques, and dropping any leftover columns with too many NaNs. To avoid data leakage, I split the data into training and testing groups based on users. I also standardized all features and used SMOTE to address class imbalances.

Feature extraction was a big part of this project. I split the time-series data into overlapping windows, with sizes from 100 to 500 samples and overlaps of 25% or 50%. From each window, I extracted 114 features, including things like mean, variance, skewness, and kurtosis. These features were consistent across both training and testing datasets. Feature selection was performed to identify the most relevant features, using mutual information and correlation analysis to reduce redundancy and improve model performance.

I used the selected features to train models like Logistic Regression, SVM, Random Forest, KNN, and XGBoost. I tuned their hyperparameters with GridSearchCV or RandomizedSearchCV and evaluated them using metrics like F1 Score, Cohen's Kappa, and Precision. SVM and Random Forest performed the best in my testing.

This project had its challenges—things like computational limits and handling redundant features—but it taught me a lot about working with time-series data and machine learning workflows. Overall, it was a excellent learning experience and I am excited to use what I learned here in future projects.

Raytracing Project

Here, I implemented a variety of features in an interactive Raytracing program. While this class was very difficult, it was also incredibly rewarding. Seeing the graphics come together over the course of the semester as I learned how to implement more and more features was so exciting.

A detailed list of all implemented functionalities:

  • Distance functions, degree/rad, basic operations (normalization, areaOfTriangle, max, min, approximatelyZero, etc.)
  • Trigonometric/Directional functions (pointOnUnitCircle, directionInRadians)
  • Quadratic Formula/Mapping
  • Vector operations (vector normalization, cosBetween, pointingVector, etc.)
  • Plane/Disk intersections
  • Raytracing
  • Lighting (ambient, diffuse, and specular color calculations, positionalLight illumination)
  • Triangles, cylinders, cones
  • Spotlights (isInSpotlightCone)
  • Textures (UV mapping)
  • Matrix Operations (Addition, multiplication, inversion)
  • 2D Transformations (Translation, rotation, scaling)
  • Pipeline Shading/Hidden surfaces: Hidden surface removal using depth Z-values (processFragment()), lighting
  • Antialiasing
  • Shadows
  • Transparent planes (alpha blending)

Spam Filter

Here, I implemented a Naive Bayes Classifier using an approach outlined in a paper written by Jonathan Lee of University of Washington. You can read the paper here

To do this, I wrote two different programs. The first program used a dataset from Kaggle to calculate the probabilities that an email is either spam or not (ham) and go through each word that appears to calculate their individual probabilities of appearing given an email is spam or ham. I then used the pickle library to save this data to my computer for easy use. The second program simply grabs all of the data from the file before taking an input string from the user and calculating whether or not it is likely spam.

While I learned how Naive Bayes works in class, I had a lot of fun implementing the logarithm-based approach he outlines on the last page. I rewrote this program to put the data in a JSON file instead of a pkl file so that all of you can play with it yourselves.
